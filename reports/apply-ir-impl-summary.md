# blitz apply IR implementation summary

Date: 2026-04-27

## Scope

Implemented the first v0.2 structured edit slice in the standalone `blitz` CLI.

## Added

- New command: `blitz apply --edit - --json [--dry-run] [--diff]`
- New module: `src/cmd_apply.zig`
- Dispatch in `src/main.zig`
- Shared exported body-range helpers in `src/edit_support.zig`

## Supported operations

- `replace_body_span`
  - exact `find`/`replace` inside target symbol body
  - `occurrence`: `only`, `first`, `last`, or positive index
  - default requires a single match
- `insert_body_span`
  - exact body anchor
  - `position`: `before` or `after`
- `wrap_body`
  - wraps full target body using `{ before, keep: "body", after, indentKeptBodyBy? }`
  - preserves function signature
- `insert_after_symbol`
  - inserts code after target symbol node

## Result schema

Returns compact JSON with:

- `status`: `applied`, `preview`, `no_changes`, or `rejected`
- `operation`, `file`, `symbol`, `language`, `dryRun`, `changed`
- `validation`: parse and match status
- `ranges`: target/body/edit byte spans
- `metrics`: file bytes, request bytes, changed bytes, wall ms
- `diffSummary`; full `diff` is opt-in and currently minimal

## Gaps

- Dedicated test module generated by subagent was discarded because it was syntactically invalid and not reliable. Current aggregate tests still pass; v0.2 should add focused `cmd_apply` tests next.
- `compose_body` not implemented in this slice.
- Diff output is compact only; full unified diff remains future work.
- JSON parsing is intentionally conservative; schema evolution should stay versioned.
